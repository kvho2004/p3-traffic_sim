    BUG PATTERN: despite both dpt storage in F3 are full, 
                the sim does not stop P1 and P2 production to start P3 production in F3

    [24797249 ms][F1-DEPT     ] Produced P1. Storage: 10/10 [24797249 ms]
    [24802831 ms][F2-DEPT     ] Produced P2. Storage: 5/5 [24802831 ms]



    Potential causes??
    - No F3 pause flags for the each dpts
    - incorrect tracking for unloading P1 units to F3

    ISSUE: Permanent Deadlock
    1. F3 dept acquires P1 lock, consumes 2 units
    2. F3 dept tries to acquire P2 lock while still holding P1 lock
    3. Ship waits on P1's not_full to unload more P1, but can't acquire P1 lock
    4. P1 storage never gets unlocked → P1 lock held forever → ship permanently blocked
    5. Both storages fill up, F3 can't continue, P3 production = 0

    SOLUTION: 
    1. Unblock all threads using pthread_cond_broadcast(&p1_s->not_full) after consuming p1 
        a. (Session 7.1 Rule 5: "Consider broadcast when you cannot determine which thread to wake up")
    2. Add pthread_mutex_unlock(&p1_s->mtx) before locking P2
