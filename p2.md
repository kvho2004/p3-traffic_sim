# The Main Scenario

## Overview

Consider the following scenario:

- There are three factories: **F1**, **F2**, and **F3**.

### Factory F1
- Produces product **P1** in a single department.
- Production rate: **1 unit every 100 ms**.
- Storage capacity: **10 units of P1**.
- If storage is full, production **pauses** until space becomes available.

### Factory F2
- Produces product **P2** in a single department.
- Production rate: **1 unit every 200 ms**.
- Storage capacity: **5 units of P2**.
- If storage is full, production **pauses** until space becomes available.

### Factory F3
- Has **two departments**.
- Each department produces **1 unit of P3** using:
  - **2 units of P1**
  - **1 unit of P2**
- Production time: **400 ms per unit**.
- Input storage:
  - **P1 storage capacity: 10 units**
  - **P2 storage capacity: 5 units**

---

## Transportation

### Transport from F1 to F3 (Ship)
- Handled by **1 ship**.
- Capacity: **5 units of P1**.
- Travel time: **250 ms each way**.
- Behavior:
  - Does **not wait** for full load (takes available items).
  - If F1 storage is empty → waits for at least **1 unit**.
  - If F3 input storage is full → waits before unloading.

### Transport from F2 to F3 (Trucks)
- Handled by **5 trucks**.
- Each truck capacity: **2 units of P2**.
- Travel time: **250 ms each way**.
- Behavior:
  - Trucks **must be fully loaded before departure**.
  - If F3 input storage is full → trucks **wait before unloading**.

# Tasks

## 1) Implementation
- Implement the scenario using **threads and locks** in **C/C++**.
- Requirements:
  - Each **department** = a thread.
  - Each **transport vehicle** = a thread.
  - Use **logging to standard output** to observe simulation behavior.

---

## 2) Deadlock Analysis
- Analyze the system to identify **potential deadlocks**.
- Provide:
  - Explanation of **where and why** deadlock can occur.
  - Description of **how** it happens in this system.
  - A **concrete solution** to prevent it.

---

## 3) Robustness Testing

The system should maintain near **100% efficiency of P3 production**.

Introduce the following disruptions:

- **P1 production stops** for several seconds.
- **P2 production stops** for several seconds.
- The **ship becomes unavailable** for several seconds.
- One **F3 department stops working** temporarily.

### Requirement
- The simulation must:
  - Handle interruptions gracefully.
  - Resume normal operation **without issues**.
